2 Git 常用命令

2.1 Git本地仓库命令

本地仓库常用命令

  • git init 新建一个仓库

  • git config –list 查看配置信息

  • git config user.name 查看用户名

  • git config user.email 查看邮箱地址

  • git config –global user.name xxxx 设置用户名

  • git config –global user.email xxxx 设置邮箱地址

  • git status 查看文件状态

  • git add 将文件的修改加入暂存区

  • git reset 将暂存区的文件取消暂存或者是切换到指定版本

    • git reset –hard 版本号回到指定版本

  • git commit -m [message] 文件名 将暂存区的文件修改提交到版本库

    • [message] 可以是一些备注信息

    • 使用 git add 命令将内容写入暂存区，git commit 命令将暂存区内容添加到本地仓库中

    • git commit -a -a 参数设置修改文件后不需要执行 git add 命令，直接来提交

  • git log 查看日志

2.2 Git远程仓库命令

  • git remote 查看远程仓库，可以查看已经关联到的远程仓库的信息。如果使用git init创建本地仓库的话，需要关联远程仓库之后才能使用该命令查到远程仓库的信息

  • git remote -v 可以查看远程仓库的详细信息

  • git remote add添加远程仓库

    • git remote add shortname url，可以使用该命令给在本地创建的仓库进行关联远程仓库，shortname一般使用origin，url使用远程仓库的url

  • git clone url 从远程仓库克隆

  • git pull 从远程仓库获取最新版本合并到本地仓库

    • git pull [shortname] [branck-name]

  • git push 推送到远程仓库

    • git push remote-name branch-name

      git push origin master一般使用这个

2.3 Git分支命令

分支是Git使用过程中很重要的概念，使用分支意味着可以把工作从开发主线上分离开来，以免影响开发主线，同一个仓库可以有多个分支，各个分支相互独立，互不干扰。通过git init命令创建本地仓库时默认会创建一个master分支

常用命令：

  • git branch 查看分支

    • git branch 列出本地所有分支
    • git branch -r 列出所有远程分支
    • git branch -a 列出本地分支和远程分支

  • git branch name 创建分支

  • git checkout name 切换分支

  • git push shortname name 推送至远程仓库分支，shortname是远程仓库的别名，name是分支名

  • git merge 其他分支名 合并分支

  • git fetch 获取远程所有分支到本地

2.4 标签命令

  • git tag 列出已有的标签
  • git tag name 创建标签
  • git push shortname name 将标签推送到远程仓库，这里的shortname指的是仓库别名，name指的是标签名
  • git checkout -b branch name 检出标签，这里的name也是指的是标签名

3.2 gitignore

在根目录下

git忽略文件：

  • 忽略操作系统自动生成的文件，比如缩略图等；
  • 忽略编译生成的中间文件、可执行文件等，也就是如果一个文件是通过另一个文件自动生成的，那自动生成的文件就没必要放进版本库，比如产生的.log日志文件；
  • 忽略你自己的带有敏感信息的配置文件，比如存放口令的配置文件。

创建仓库时可以选择自动生成gitignore文件
